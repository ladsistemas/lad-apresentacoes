Operação no KDS

A cozinha acompanha e gerencia a produção pelo KDS.

Ponto natural para acionar o entregador no momento certo.

Objetivo: usar a Husky como malha de entrega para as lojas que não têm logística própria — de forma confiável e com visibilidade do status.

Quem é a Husky

Logística de entrega (last-mile)

A Husky é uma plataforma de roteirização e gestão de entregadores. O estabelecimento envia o pedido e a Husky cuida da coleta e entrega até o cliente.

1. Cria a entrega

Enviamos endereço, valor e forma de pagamento via API.

2. Aciona entregador

A Husky aloca o entregador e gerencia a rota.

3. Acompanha status

Atualiza o andamento até a entrega finalizada.

Comunicação por API REST com autenticação por token e suporte a ambiente de testes (sandbox).

Pronto hoje

Envio automático da entrega

Criação da entrega

Ao gravar um pedido de delivery, o sistema cria a entrega na Husky automaticamente (endpoint createOrderList).

Habilitação por estabelecimento

Cada empresa tem seu token Husky. Sem token, a integração fica desligada.

Dois pontos de origem

Funciona tanto para pedidos do app do cliente quanto para o delivery lançado no PDV.

Dados enviados

Cliente, telefone, endereço completo, bairro/cidade/UF, valor e forma de pagamento.

  Sem acompanhamento

Depois de criar a entrega, não recebemos status de volta. Não sabemos se foi coletado, está a caminho ou entregue.

  Sem "pedido pronto"

Marcar o pedido como pronto no KDS não avisa a Husky. O entregador não é acionado no momento ideal.

São exatamente essas duas lacunas que as melhorias a seguir resolvem.

  Timing certo

  Encaixe natural no KDS

Melhoria 3

Rastreio e dados do entregador

Guardar o código da entrega

Hoje não guardamos o código de rastreio que a Husky devolve. Ele é a base para status, "pedido pronto" e cancelamento.

Entregador identificado

Exibir nome, foto e telefone do entregador no acompanhamento do pedido.

Cancelamento sincronizado

Pedido cancelado no LADFood reflete na Husky, evitando deslocamento desnecessário do entregador.

Previsão de chegada

O webhook traz o ETA (tempo estimado), exibível ao operador e ao cliente.

Saúde da integração

Correções no fluxo atual

Ajustes técnicos para deixar o que já existe mais confiável (transparentes para o cliente).

  Ambiente de testes

Apontar para a URL de sandbox correta em homologação (hoje testes vão para produção).

  Envio assíncrono

Hoje o envio é síncrono e pode travar a gravação do pedido se a Husky demorar. Tornar assíncrono + reenvio.

  Forma de pagamento

Mapear as formas do LADFood para os valores aceitos pela Husky (ONLINE, CASH, TERMINAL).

  Endereço / geolocalização

Garantir geocodificação ou lat/lng para a Husky localizar a entrega; preencher o número do endereço.

Decisão importante

Quando usar a Husky? (por canal)

Nem todo pedido deve ir para a Husky. Cada canal pode ter logística própria ou entrega do estabelecimento.

CanalEntrega da plataformaEntrega do estabelecimento
iFoodiFood entrega → não enviaLoja entrega → via Husky
99Food99Food entrega → não enviaLoja entrega → via Husky
Anota.ai—Loja entrega → via Husky

A própria plataforma já informa quem entrega. A proposta é configurar por loja/canal qual usa Husky, e respeitar automaticamente quando a entrega é da plataforma.
